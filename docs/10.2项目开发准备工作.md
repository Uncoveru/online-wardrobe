## 10.2.1 项目开发及运行环境

为了让读者在学习本项目的开发的过程中，避免出现不必要的错误，下面对网上衣橱的开发及运行所需的环境和相关软件进行介绍，具体如下。

## 10.2.2 数据库设计

由于本项目用到的数据需要永久存储，所以在开始开发项目之前要进行数据库设计。数据库是根据程序的要求及其实现功能设计的，数据库设计的合理性将直接影响程序的开发过程。下面对网上衣橱项目的数据库设计进行讲解。

本项目涉及的实体包括用户、服装、服装的尺码、服装的类型、购物车和订单，因此需要在数据库中创建实体对应的数据表，具体如下。

(1) 用户表（t_user）

t_user 表用于保存网上衣橱系统的前台用户和后台管理员的信息，其表结构如表 10-1 所示。

## 10.2.3 项目工程搭建

在开发功能模块之前，应该先进行项目环境及项目框架的搭建工作。下面分步骤讲解在正式开发功能模块前应做的准备工作。

(1) 创建数据库表

在 MySQL 数据库中创建一个名称为 wardrobe 的数据库，并根据 10.2.2 小节中的表结构在 wardrobe 数据库中创建相应的表。由于测试功能时需要用到一些初始数据，本书的配套资源中提供了该项目的数据库所对应的 SQL 文件，读者可以直接运行 SQL 文件进行表的创建和初始数据的添加，以便后续测试。

(2) 准备前端项目

本网上衣橱系统为前后端分离的设计，为了使读者能够专注于后端 Web 程序的开发，本书的配套资源中提供了该项目所需的 2 个前端项目的压缩文件，其中名称为 wardrobe_user.zip 的文件为前台用户端的项目文件，名称为 wardrobe_admin.zip 的文件为后台管理端的项目文件。读者可以将这两个文件解压缩到没有中文的路径下，然后运行即可。需要注意的是，本书提供的 wardrobe_user 项目和 wardrobe_admin 项目的运行端口号分别为 7070 和 7071，读者需要确保自己的计算机中这两个端口号没有被占用，如果已被占用，可以将其修改为其他未占用的端口号。

(3) 导入后端初始工程

为了便于后续功能模块的开发，本章提供了一个初始工程，初始工程中根据项目功能创建了对应的包用于存放项目源代码，并且提供了项目的依赖信息、工具类、初始的服装图片。在 IDEA 中导入初始工程，可以看到初始工程的主体结构如图 10-10 所示。

**图 10-10 wardrobe_back 的项目结构**

下面对项目中的包进行说明。

1.  controller：用于存放项目的 Servlet 类，这些类中根据请求路径调用 service 层类中对应的方法处理业务请求，并返回响应。
2.  dao：用于存放与数据库进行交互的类，这些类中编写与数据库交互的代码，包括 SQL 语句、结果处理等。
3.  filter：用于存放过滤器类。
4.  model：用于存放实体类。为了方便后续项目功能的实现，导入的项目中提供了项目中所用的实体类，这些实体类对应数据库中的数据表，分别为 User 类（用户类）、Clothes 类（服装类）、Size 类（服装的尺码类）、Type 类（服装的类型类）、Cart 类（购物车类）、Order 类（订单类）。
5.  service：用于存放业务逻辑类，这些类主要用于编写业务逻辑，通过调用 dao 层类中对应的方法处理对应的业务需求。
6.  utils：用于存放工具类，初始项目中提供了 DruidUtils 类和 TestUtils 类，这两个工具类的说明如下。
    - LocalDateTimeUtil 类：LocalDateTimeUtil 类为日期时间工具类，用于将日期时间对象与字符串格式的日期时间相互转换。
    - DruidUtils 类：DruidUtils 类为数据库连接池工具类，用于初始化连接池并提供获取连接池对象以及释放资源。
    - TestUtils 类：TestUtils 类为 JWT 工具类，用于简化 JWT 的生成、解析、验证等过程，使得开发者能够更专注于业务逻辑的实现。JWT（JSON Web Token）是一种跨域认证的解决方案，主要用于在用户登录验证和身份识别中发挥作用。在本项目中，由于我们采用了前后端分离的开发模式，面临着跨域认证问题，而传统的 Session 机制无法满足跨域需求，因此选择使用 JWT。JWT 能够生成包含用户信息的 token，每次请求服务器时只需要使用保存的密钥验证 token 的正确性，不用再保存任何 session 数据，就可以实现认证流程。
